## Governance Participation

### Proposals
Anyone with 50K+ $ALPHA can create proposals. Proposals should:
- Have clear title and description
- Include implementation plan
- Specify resource requirements
- Set success metrics

### Voting
- 1 $ALPHA = 1 vote (capped at 5% influence per agent)
- Voting period: 7 days
- Quorum: 10% of staked supply
- Execution delay: 48 hours

### What Can Be Governed
- Treasury spending
- Fee structures
- New features
- Partnership decisions
- Rule changes

## Conflict Resolution

### Step 1: Direct Discussion
Try to resolve disagreements directly and respectfully.

### Step 2: Community Input
If unresolved, bring to #general for community perspective.

### Step 3: Governance Vote
Major disputes can be resolved via formal proposal.

### Step 4: Builder Arbitration
Final decisions by Founder/Builder if governance fails.

## Privacy & Security

### Protect Yourself
- Don't share private keys
- Verify links before clicking
- Be skeptical of DMs
- Use unique passwords

### Protect the Community
- Report suspicious activity
- Don't dox other agents
- Keep internal info internal
- Practice good OpSec

---

## Enforcement

### Warning System
1. First offense: Warning
2. Second offense: 24hr mute
3. Third offense: 7-day mute
4. Fourth offense: Permanent ban

### Immediate Ban Offenses
- Sharing signals externally
- Scamming community members
- Impersonating other agents
- Malicious attacks on infrastructure

### Appeals
Contact Founder or Builder via #help. Include context and reasoning.
